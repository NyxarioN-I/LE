Lesson 04

إعداد VirtualBox

Configuring VirtualBox

لماذا نحتاج إعداد الـ VM؟

بعد تثبيت Kali Linux، لازم نعدّ بعض الإعدادات عشان الـ VM يشتغل بشكل أفضل وسريع.

الإعدادات المهمة

1. System (النظام)

اضغط على VM ثم Settings ثم تبويب System:

Motherboard (اللوحة الأم)

  • Boot Order: ضع Hard Disk في الأعلى (أولاً)
  • Base Memory: 4096 MB (4GB) كحد أدنى

Processor (المعالج)

  • Processors: 2 معالج كحد أدنى
  • إذا كان جهازك قوي، زِد العدد

2. Display (العرض)

  • Video Memory: 128 MB
  • Graphics Controller: VMSVGA

3. Network (الشبكة)

هذا أهم إعداد!

  • Attached to: Bridged Adapter
  • Name: اختر واجهة الشبكة الخاصة بك
لماذا Bridged Adapter؟

هذا الإعداد يجعل الـ VM يظهر كجهاز مستقل على الشبكة. يعني يقدر يحصل على IP من الراوتر ويتشابك مع الأجهزة الأخرى في الشبكة. هذا ضروري لاختبار الشبكات.

الإعدادات خطوة بخطوة

System ├── Motherboard │ ├── Boot Order: Hard Disk أولاً │ └── Base Memory: 4096 MB ├── Processor │ └── Processors: 2 ├── Display │ ├── Video Memory: 128 MB │ └── Graphics Controller: VMSVGA └── Network └── Adapter 1 └── Attached to: Bridged Adapter

إعدادات إضافية

Shared Folders (مجلدات مشتركة)

تقدر تشارك مجلد من جهازك الحقيقي مع الـ VM:

  1. اذهب لـ Settings ثم Shared Folders
  2. اضغط على أيقونة المجلد (+)
  3. حدد المجلد من جهازك
  4. حدد "Auto-mount"

USB Settings

إذا كنت تحتاج توصيل أجهزة USB بالـ VM:

  1. اذهب لـ Settings ثم USB
  2. اختر "USB 2.0 (EHCI) Controller" أو "USB 3.0"

التشغيل الأول

بعد الإعداد، شغّل الـ VM:

  1. اضغط "Start" لتشغيل الـ VM
  2. ستظهر لك شاشة Kali Linux
  3. سجّل الدخول
  4. افتح Terminal
📝 تمرين 4: إعداد الـ VM
  1. افتح إعدادات الـ VM الخاص بك
  2. عدّل الإعدادات كما هو موضح أعلاه
  3. تأكد أن Boot Order يبدأ من Hard Disk
  4. تأكد أن Base Memory 4GB على الأقل
  5. تأكد أن Network على Bridged Adapter
  6. شغّل الـ VM وتأكد أنه يعمل بشكل سريع
💡 التحقق من الإعدادات

داخل Kali Linux، افتح Terminal واكتب:

free -h

تأكد أن الذاكرة تقريباً 4GB.

ip a

تأكد أن عندك عنوان IP من الشبكة (ليس 127.0.0.1).
