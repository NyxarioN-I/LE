Lesson 09

أوامر الشبكة

Common Network Commands

أوامر عرض معلومات الشبكة

1. ip a أو ifconfig - عرض عناوين IP

تُستخدم لعرض واجهات الشبكة وعناوين IP المخصصة لها.

# الطريقة الجديدة (الموصى بها) ip a # الطريقة القديمة ifconfig

شرح المخرجات

2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> link/ether 08:00:27:xx:xx:xx inet 192.168.1.100/24 brd 192.168.1.255 scope global eth0 inet6 fe80::a00:27ff:xxxx:xxxx/64 scope link التفسير: - eth0 = اسم الواجهة - UP = الواجهة مفعلة - inet = عنوان IPv4 - /24 = ك水准 الشبكة (Subnet Mask) - brd = عنوان البث (Broadcast)

2. ping - فحص الاتصال

تُستخدم للتحقق من الاتصال بجهاز آخر في الشبكة.

# فحص الاتصال بجهاز ping 192.168.1.1 # إرسال 4 حزم فقط (مثل Windows) ping -c 4 google.com # فحص بمنفذ محدد ping -p 80 google.com # التوقف بعد وقت محدد ping -c 10 192.168.1.1
معلومة مهمة

بعض الأجهزة توقف ICMP ولن ترد على ping. هذا لا يعني أنها غير متصلة. استخدم nmap كبديل.

3. ip r - جدول التوجيه

تُعرض جدول التوجيه (Routing Table) الذي يُحدد كيفية وصول حركة المرور.

# عرض جدول التوجيه ip r # أو route # المخرجات: default via 192.168.1.1 dev eth0 192.168.1.0/24 dev eth0 scope link # التفسير: # - الجهة الافتراضية: 192.168.1.1 (الراوتر) # - الشبكة المحلية: 192.168.1.0/24

4. arp -a - الجدول المرسال

تُعرض ارتباط عناوين IP بعناوين MAC.

# عرض ARP Table arp -a # أو ip n # المخرجات تُظهر: # - عنوان IP ↔ عنوان MAC # - الارتباط بينهما

5. iwconfig - معلومات الشبكة اللاسلكية

# عرض معلومات الشبكات اللاسلكية iwconfig

لماذا أوامر الشبكة مهمة للهاكرز؟

  • اكتشاف الأجهزة: معرفة الأجهزة المتصلة في الشبكة
  • فحص المنافذ: معرفة الخدمات تعمل على أي منافذ
  • تتبع حركة المرور: فهم كيفية تدفق البيانات
  • اكتشاف الشبكات المخفية: البحث عن شبكات غير مرئية
📝 تمرين 9: تمارين الشبكة
  1. شغّل ip a وحدد عنوان IP الخاص بك
  2. شغّل ping -c 4 google.com لفحص الاتصال بالإنترنت
  3. شغّل ip r لمعرفة عنوان الراوتر
  4. شغّل arp -a لرؤية الأجهزة المتصلة
  5. شغّل ping على عنوان الراوتر
