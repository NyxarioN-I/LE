Lesson 11

بدء وإيقاف الخدمات

Starting and Stopping Services

ما هي الخدمات؟

الخدمات (Services) في Linux هي برامج تعمل بالخلفية وتوفّر وظائف معينة. مثل:

  • خادم الويب: Apache أو Nginx
  • SSH: للوصول عن بُعد
  • قاعدة البيانات: MySQL أو PostgreSQL
  • جدار الحماية: UFW أو Firewalld

أمر systemctl

systemctl هو الأمر الأساسي لإدارة الخدمات في Linux الحديثة.

الأوامر الأساسية

# بدء تشغيل خدمة sudo systemctl start [اسم-الخدمة] # إيقاف خدمة sudo systemctl stop [اسم-الخدمة] # إعادة تشغيل خدمة sudo systemctl restart [اسم-الخدمة] # عرض حالة خدمة sudo systemctl status [اسم-الخدمة] # تفعيل التشغيل التلقائي مع النظام sudo systemctl enable [اسم-الخدمة] # تعطيل التشغيل التلقائي sudo systemctl disable [اسم-الخدمة] # عرض كل الخدمات النشطة systemctl list-units --type=service

أمثلة عملية

1. خادم الويب Apache

# بدء تشغيل Apache sudo systemctl start apache2 # التحقق من الحالة sudo systemctl status apache2 # إيقاف Apache sudo systemctl stop apache2 # إعادة تشغيل Apache sudo systemctl restart apache2 # تفعيل التشغيل التلقائي sudo systemctl enable apache2

2. خدمة SSH

# بدء تشغيل SSH sudo systemctl start ssh # التحقق من الحالة sudo systemctl status ssh # تفعيل التشغيل التلقائي sudo systemctl enable ssh

3. قاعدة البيانات MySQL

# بدء تشغيل MySQL sudo systemctl start mysql # التحقق من الحالة sudo systemctl status mysql

قراءة مخرجات status

عندما تكتب systemctl status، سترى something مثل:

● apache2.service - The Apache HTTP Server Loaded: loaded (/lib/systemd/system/apache2.service; enabled; vendor preset: enabled) Active: active (running) since Mon 2024-01-15 10:30:00 UTC; 5min ago التفسير: - Loaded: تم تحميل الخدمة - enabled: ستبدأ تلقائياً مع النظام - Active: active (running) = تعمل حالياً - منذ متى وهي تعمل

📝 تمرين 11: تمارين الخدمات
  1. شغّل أمر sudo systemctl status apache2 واقرأ الحالة
  2. إذا لم يكن Apache مثبّتاً، ثبّته بـ sudo apt install apache2
  3. شغّل Apache ثم تحقق من أنه يعمل
  4. افتح المتصفح واكتب http://localhost لرؤية صفحة Apache
  5. أوقف Apache وتأكد أنه توقف
