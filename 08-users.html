Lesson 08

المستخدمون والصلاحيات

Users and Privileges

أنواع المستخدمين

في Linux، هناك أنواع مختلفة من المستخدمين، لكل نوع صلاحيات مختلفة.

النوع الصلاحيات المثال
Rootصلاحيات كاملة على النظام بالكاملroot
Regular Userصلاحيات محدودةkali, hacker
System Userلتشغيل الخدمات فقطwww-data, mysql

أنواع الصلاحيات

كل ملف ومجلد في Linux له صلاحيات تحدد من يمكنه قراءته أو كتابته أو تشغيله.

الأنواع الثلاثة للصلاحيات

r = Read (قراءة) = 4 w = Write (كتابة) = 2 x = Execute (تنفيذ) = 1 - = No permission = 0

من يمكنه الصلاحيات؟

- rwx r-x r-- 1 user group 4096 file.txt │││ │││ │││ │││ │││ └┴┴── Others (الجميع) │││ └┴┴────── Group (المجموعة) └┴┴────────── User/Owner (المالك)

أمثلة على الصلاحيات

الرقم المعنى الشرح
755rwxr-xr-xالمالك: كامل، المجموعة والجميع: قراءة وتنفيذ
644rw-r--r--المالك: قراءة وكتابة، الآخرون: قراءة فقط
777rwxrwxrwxالكل: كامل الصلاحيات (خطير!)
444r--r--r--الكل: قراءة فقط

أوامر إدارة المستخدمين

# عرض المستخدم الحالي whoami # عرض معلومات المستخدم id # إضافة مستخدم جديد sudo adduser newuser # إضافة مستخدم لمجموعة sudo usermod -aG sudo newuser # تغيير كلمة المرور passwd # حذف مستخدم sudo deluser newuser # عرض كل المستخدمين cat /etc/passwd # عرض المجموعات cat /etc/group

تغيير الصلاحيات chmod

chmod اختصار لـ Change Mode. يُستخدم لتغيير صلاحيات الملفات والمجلدات.

الطريقة الرقمية

# جعل الملف قابل للتنفيذ chmod +x script.sh # إضافة تنفيذ للمالك chmod u+x file.txt # إضافة كتابة للمجموعة chmod g+w file.txt # إزالة قراءة عن الجميع chmod o-r file.txt # استخدام الأرقام chmod 755 file.txt # rwxr-xr-x chmod 644 file.txt # rw-r--r-- chmod 777 file.txt # rwxrwxrwx (danger!) chmod 444 file.txt # r--r--r-- # تغيير الصلاحيات لمجلد وتحته chmod -R 755 directory/

الطريقة الرمزية

# u = user (المالك) # g = group (المجموعة) # o = others (الجميع) # a = all (الكل) chmod u+x file.txt # إضافة تنفيذ للمالك chmod g+w file.txt # إضافة كتابة للمجموعة chmod o-r file.txt # إزالة قراءة عن الجميع chmod a+r file.txt # إضافة قراءة للكل

تغيير المالك chown

# تغيير مالك الملف sudo chown user:group file.txt # تغيير مالك مجلد وتحته sudo chown -R user:group directory/ # تغيير المالك فقط sudo chown user file.txt # تغيير المجموعة فقط sudo chgrp group file.txt
تحذير

لا تستخدم chmod 777 إلا عند الضرورة القصوى. هذا يعطي كل الصلاحيات للجميع وهو خطير على الأمان!

📝 تمرين 8: تمارين المستخدمين والصلاحيات
  1. شغّل whoami وid لمعرفة معلوماتك
  2. أنشئ مستخدم جديد باسم testuser
  3. أنشئ ملفاً وغيّر صلاحياته إلى 755
  4. أنشئ ملفاً واجعله للقراءة فقط 444
  5. جرّب حذف الملف المحمي وشاهد الخطأ
💡 الحل
whoami id sudo adduser testuser touch testfile1.txt chmod 755 testfile1.txt ls -l testfile1.txt touch testfile2.txt chmod 444 testfile2.txt ls -l testfile2.txt rm testfile2.txt # سترى: rm: remove write-protected regular empty file 'testfile2.txt'?
