أمر Sudo

Sudo Overview

ما هو Sudo؟

Sudo اختصار لـ Super User DO (نفّذ كمدير). هذا الأمر يعطيك صلاحيات المدير (Root) مؤقتاً لتنفيذ أوامر معينة.

لماذا نحتاج Sudo؟

في Linux، بعض الأوامر تحتاج صلاحيات خاصة لتنفيذها. بدون Sudo، سترى رسالة خطأ مثل:

sudo: unable to initialize policy plugin

أمثلة على عمليات تحتاج Sudo

  • تنزيل برامج جديدة
  • تعديل ملفات النظام
  • تغيير إعدادات الشبكة
  • بدء أو إيقاف الخدمات
  • تعديل صلاحيات الملفات

كيفية استخدام Sudo

# الصيغة الأساسية sudo [الأمر] # مثال: تحديث النظام sudo apt update # مثال: تثبيت برنامج sudo apt install nmap # مثال: الدخول كمدير كامل sudo su # الخروج من وضع المدير exit

أوامر Sudo المفيدة

# عرض صلاحيات المستخدم الحالي sudo -l # الدخول كمدير كامل (غير موصى به) sudo su # تنفيذ أمر بصلاحيات مستخدم آخر sudo -u username command # عرض معلومات sudoers sudo cat /etc/sudoers # تعديل sudoers (بمعدل visudo) sudo visudo
تحذير مهم

لا تستخدم sudo su إلا عند الضرورة القصوى. العمل بصلاحيات root قد يسبب مشاكل إذا كتبت أمر خطأ. الأفضل استخدام sudo قبل كل أمر يحتاج صلاحيات.

النصيحة الذهبية

قاعدة مهمة

إذا كنت تتردد: هل أحتاج sudo أم لا؟ الجواب通常是 نعم. إذا طلب منك كلمة مرور، فأنت تحتاج sudo.

📝 تمرين 6: تمارين Sudo
  1. شغّل أمر apt update بدون sudo وشاهد الخطأ
  2. شغّل نفس الأمر مع sudo apt update وتأكد أنه يعمل
  3. شغّل sudo -l لعرض صلاحياتك
  4. شغّل whoami ثم sudo whoami وقارن النتائج
